# 誤り訂正について

### 誤り訂正とは？

データを元に作られた誤り訂正コードをデータに付加することで、一部きちんと読み取らなくても、だいたい読み取れていればデータの内容がわかるようになる。

### 簡単な例

英語のリスニングは、だいたい2回流れますよね。2回目の音声は、考えようによっては「誤り訂正」とみなすことができます。

というのも、1回目のリスニングで必要な情報は全て受け取っているわけです。でも、うまくききとれなかったりして、一部よくわからないところが出てくると思います。
そこで、同じ音声を聞くことで、1回目の聞き取った内容を修正して、より内容を理解するわけです。

これと同じように、二次元コードに限らず全ての情報は、"同じような" 内容を複数回流すことで、情報がきちんと届くように工夫されています。

### 二次元コードの場合

実際には、複雑な方法で元のデータからうまく誤り訂正コードを作ることで、効率的にデータの復元ができるようになっています。

例に挙げている二次元コードだと、データコードが 128bit に対し、誤り訂正コードは 80 bit であり、
データが 15% ほど間違っていても、復元して元のデータに戻すことができます。
